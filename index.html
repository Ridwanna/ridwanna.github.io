								<p>..contents..</p>,

Types of Element

Every HTML element has a default display value depending on what type of element it is. The default display value for most elements is block or inline

Block-level Elements

A block-level element always starts on a new line and takes up the full width available (stretches out to the left and right as far as it can). The <div> element is a block-level element. 

									
								Examples are: 
									<video>, <article>, 
									<aside>, <section>, 
									<canvas>, “<nav>, 
									<div>, <footer>, 
									<header>, <p>, etc
Inline Elements

An inline element does not start on a new line and only takes up as much width as necessary. This is an inline <span>..conetents..</span> element inside a paragraph. 

									
								Examples are: 
									<abbr>, <img>, 
									<b>, <b>, 
									<select>, <big>, 
									<button>, <a> 
									<label>, <input>, 
									<script>, <style> etc

DIV Element: The <div> element is often used as a container for other HTML elements. The <div> element has no required attributes, but style, class and id are common. When used together with CSS, the <div> element can be used to style blocks of content.


SPAN Element: The <span> element is often used as a container for some text. The <span> element has no required attributes, but style, class and id are common. When used together with CSS, the <span> element can be used to style parts of the text.


HTML Tags

These are the hidden keywords within a web page that define how your web browser must format and display the contents. Most tags must have two parts, an opening and a closing part.

							
							Examples are:
								<html> is the opening tag and </html> is the closing tag.
								<html>, <h1>, <ol>, 
								<button>, <head>, </html>,
								</h1>, </body>, </head>,  
								</p>, <button>, etc

Self-closing tags don't require an ending tag.

							
									<br>, <hr>,
									 <img>, <input>, 
									 <link>, <meta>, etc

An Attribute

This defines a property for an element, consists of an attribute/value pair, and appears within the element's start tag. ... The most popular misuse of the term "tag" is referring to alt attributes as "alt tags". There is no such thing in HTML. Alt is an attribute, not a tag.

Note: Tags are used to mark up the start and end of an HTML element. An attribute defines a property for an element. They are also extra bits of information. Attributes appear inside the opening tag and their values sit inside quotation marks. They look something like: <tag attribute = "value">

Image Tag

In HTML, images are defined with the img tag. The img tag is empty, it contains attributes only, and does not have a closing tag. The src attribute specifies the URL (web address) of the image and it is very compulsory. Other attributes are: alt, length, width, style

Use the <map> tag to define an image-map. An image-map is an image with clickable areas.

							
								Syntax: <img src=" " />

Video Tag

To show a video in HTML, use the <video> element. Supported formats are MP4, WebM, Ogg. 

							
							Syntax: <video width="320" height="240" controls> 
<source src="movie.mp4" type="video/mp4" >

The controls attribute adds video controls, like play, pause, and volume. It is a good idea to always include width and height attributes. If height and width are not set, the page might flicker while the video loads.

Helper applications (plug-ins)

They are computer programs that extend the standard functionality of a web browser. Examples of well-known plug-ins are Java applets. Plug-ins can be added to web pages with the <object> tag or the <embed> tag

Plug-ins can be used for many purposes: display maps, scan for viruses, verify your bank id, etc. The <object> element is supported by all browsers. The <object> element defines an embedded object within an HTML document. It is used to embed plug-ins (like Java applets, PDF readers, Flash Players, html doc) in web pages.4

LIST

Unordered List

An Unordered List starts with the <ul> tag. Each list item starts with the <li> tag. The list items will be marked with bullets (small black circles) by default. The CSS list-style-type property is used to define the style of the list item marker: Disc (default), Circle, Square, None. 

							
							Syntax:
								<ul>
									<li>...Content...</li>
									<li>...Content...</li>
								</ul>

Ordered List

An Ordered List starts with the <ol> tag. Each list item starts with the <li> tag. The list items will be marked with numbers by default. The type attribute of the <lo> tag, defines the type of the list item marker: 1, A, a, I, i 

							
							Syntax:
								<ol>
									<li>...Content...</li>
									<li>...Content...</li>
								</ol>

Descriptive List

HTML also supports description lists. A description list is a list of terms, with a description of each term. The <dl> tag defines the description list, the <dt> tag defines the term (name), and the <dd> tag describes each term: 

						
						Syntax:
							<dl>
								<dt>....</dt>
									<dd>...Content...</dd>
								<dt>....</dt>
									<dd>...Content...</dd>
							</dl>

BOILERPLATE

This is the basic structure of an html document/page. HTML Living Standard defines that the <html>, <head> and <body> tags may be safely omitted under most circumstances. The <meta charset="UTF-8"> tag may also be omitted if the Web server is properly configured to send the character encoding along with the content type. Google’s HTML/CSS style guide recommends that all optional tags be omitted, resulting in a much more compact boilerplate 

					
					Syntax:
						<!DOCTYPE html>
							<html>
								<head>
									<meta charset="UTF-8">
									<title> .... </title>
								</head>
								<body>
									.... contents
								</body>
							</html>
